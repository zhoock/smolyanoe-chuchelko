# 📋 Подробная инструкция: Настройка переменных окружения в Netlify

## 🎯 Цель

Настроить переменные окружения для работы платежной системы с базой данных PostgreSQL и шифрованием.

## 📍 Шаг 1: Откройте Netlify Dashboard

1. Перейдите на https://app.netlify.com
2. Войдите в свой аккаунт
3. Найдите проект **smolyanoechuchelko** (или ваш проект)
4. Нажмите на название проекта

## 📍 Шаг 2: Перейдите в настройки переменных окружения

### Вариант A: Через левое меню (рекомендуется)

1. В **левом вертикальном меню** найдите пункт **"Project configuration"** (⚙️)
   - Обычно это второй пункт ниже "Project overview"
2. Нажмите на **"Project configuration"**
3. В открывшемся подменю найдите **"Environment variables"**
4. Нажмите на **"Environment variables"**

### Вариант B: Через верхнее меню

1. В верхнем меню найдите **"Site settings"**
2. В левом меню найдите **"Environment variables"**
3. Нажмите на **"Environment variables"**

## 📍 Шаг 3: Добавьте переменную DATABASE_URL

### 3.1. Нажмите "Add a variable"

На странице "Environment variables" нажмите кнопку **"Add a variable"** (обычно вверху справа).

### 3.2. Заполните форму

**Key (Имя переменной):**

```
DATABASE_URL
```

**Value (Значение):**

```
postgresql://postgres.jhpvetvfnsklpwswadle:[YOUR-PASSWORD]@aws-1-ap-south-1.pooler.supabase.com:5432/postgres
```

**⚠️ ВАЖНО:** Замените `[YOUR-PASSWORD]` на ваш пароль базы данных Supabase.

**Scope (Область применения):**

- Выберите **"Production"** (для production окружения)
- Или **"All scopes"** (для всех окружений)

### 3.3. Сохраните

Нажмите кнопку **"Save"** или **"Add variable"**.

## 📍 Шаг 4: Добавьте переменную ENCRYPTION_KEY

### 4.1. Снова нажмите "Add a variable"

### 4.2. Заполните форму

**Key (Имя переменной):**

```
ENCRYPTION_KEY
```

**Value (Значение):**

```
[YOUR-ENCRYPTION-KEY]
```

**⚠️ ВАЖНО:** Замените `[YOUR-ENCRYPTION-KEY]` на ваш ключ шифрования (должен быть длинной строкой в base64).

**Scope (Область применения):**

- Выберите **"Production"** (для production окружения)
- Или **"All scopes"** (для всех окружений)

### 4.3. Сохраните

Нажмите кнопку **"Save"** или **"Add variable"**.

## 📍 Шаг 5: (Опционально) Переменные ЮKassa **без** глобального магазина

Приложение оплачивает **через креды продавца из БД** (`user_payment_settings`). В Netlify достаточно при необходимости:

### 5.1. YOOKASSA_API_URL (опционально)

**Key:**

```
YOOKASSA_API_URL
```

**Value:**

```
https://api.yookassa.ru/v3/payments
```

**Legacy:** `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY` приложение для создания платежей **не читает**. Их можно не добавлять; старые значения можно удалить.

## 📍 Шаг 6: Передеплойте проект

**КРИТИЧЕСКИ ВАЖНО:** После добавления переменных окружения проект нужно передеплоить!

### 6.1. Вернитесь на главную страницу проекта

Нажмите на **"Project overview"** в левом меню.

### 6.2. Запустите новый деплой

**Вариант A: Через кнопку "Trigger deploy"**

1. Найдите раздел **"Production deploys"** на странице
2. Нажмите кнопку **"Trigger deploy"**
3. Выберите **"Deploy site"**

**Вариант B: Через меню "Deploys"**

1. В верхнем меню найдите **"Deploys"**
2. Нажмите на **"Deploys"**
3. Нажмите кнопку **"Trigger deploy"** → **"Deploy site"**

### 6.3. Дождитесь завершения деплоя

- Статус деплоя отображается в разделе **"Deploys"**
- Дождитесь статуса **"Published"** (зелёная галочка)

## ✅ Проверка

### 1. Проверьте переменные окружения

1. Вернитесь в **"Environment variables"**
2. Убедитесь, что видны обе переменные:
   - ✅ `DATABASE_URL`
   - ✅ `ENCRYPTION_KEY`

### 2. Проверьте логи функций

1. Перейдите в **"Deploys"** → выберите последний деплой
2. Откройте **"Functions"** → **"Logs"**
3. Проверьте, что нет ошибок подключения к базе данных

### 3. Протестируйте платежную систему

1. Откройте ваш сайт
2. Попробуйте создать тестовый платеж
3. Проверьте, что всё работает корректно

## 🔒 Безопасность

### ✅ Что правильно:

- ✅ Переменные окружения хранятся в зашифрованном виде в Netlify
- ✅ Они доступны только в runtime (не в коде)
- ✅ Разные ключи для разных окружений (Production, Deploy previews)

### ⚠️ Что НЕ делать:

- ❌ НЕ коммитьте переменные окружения в Git
- ❌ НЕ делитесь ключами в открытом виде
- ❌ НЕ используйте один и тот же `ENCRYPTION_KEY` для разных проектов

## 📊 Итоговый список переменных

После настройки у вас должно быть:

| Переменная         | Обязательно | Scope      | Пример значения      |
| ------------------ | ----------- | ---------- | -------------------- |
| `DATABASE_URL`     | ✅ Да       | Production | `postgresql://...`   |
| `ENCRYPTION_KEY`   | ✅ Да       | Production | `MLjc/tUOfSm/...`    |
| `YOOKASSA_API_URL` | ❌ Нет      | любой      | см. код по умолчанию |

**Не используются для оплаты (legacy):** `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`.

## 🆘 Проблемы и решения

### Проблема: "Variable not found"

**Решение:**

- Убедитесь, что переменная добавлена в правильный Scope (Production)
- Передеплойте проект после добавления переменных

### Проблема: "Database connection failed"

**Решение:**

- Проверьте правильность `DATABASE_URL` (включая пароль)
- Убедитесь, что база данных доступна из интернета
- Проверьте, что используется правильный порт (5432 для Supabase)

### Проблема: "Encryption key invalid"

**Решение:**

- Убедитесь, что `ENCRYPTION_KEY` скопирован полностью (включая `=`)
- Проверьте, что нет лишних пробелов в начале или конце
